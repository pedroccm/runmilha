### Build Fix — tsconfig excluding CMS
- Root `tsconfig.json` had `include: ["**/*.ts", "**/*.tsx"]` which pulled in `/cms` files during build
- CMS `plans/page.tsx` imported `formatCurrency` from its own `@/lib/utils`, but root build resolved `@/` to root `src/`, causing "Module has no exported member 'formatCurrency'" error
- **Fix:** Added `"cms"` to `exclude` array in root `tsconfig.json`
- This fixed 3 consecutive failed Netlify deploys

### Strava Webhook Fix
- **Problem 1:** Webhook handler used `processActivityEvent(event).catch(console.error)` — fire-and-forget async. On Netlify serverless, the function is killed after responding, so processing never completed.
- **Fix:** Changed to `await processActivityEvent(event)` — synchronous processing before responding 200.
- **Problem 2:** "Walk" activity type was not in `SUPPORTED_ACTIVITY_TYPES`. User's test walk was silently ignored.
- **Fix:** Added "Walk" and "Hike" to the supported types array in `src/lib/constants.ts`.
- After fixes, user's "Caminhada noturna" (22m walk) was successfully processed: activity saved to `rm_activities`, 0.01 milhas credited.

### Broken Reward Images

- Package name: `com.runmilha.app`

### Architecture
- **Auth:** Supabase client with `expo-secure-store` for token persistence
- **Navigation:** Bottom tab bar with 5 tabs (Home, Activities, Wallet, Marketplace, Profile)
- **State:** React Context for auth (AuthProvider wraps app)
- **Data:** Direct Supabase queries from each screen, pull-to-refresh on all lists
